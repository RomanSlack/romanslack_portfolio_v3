## Writing rules (whenever you write anything user-facing in this repo)

- NEVER use em dashes. Not in copy, titles, descriptions, JSON-LD, commit messages, anything. Use commas, parentheses, or separate sentences instead.
- Any prose meant to sound like Roman (essays, monologues, posts, project blurbs, talks) has to read like him, not like generic AI. Read `roman_slack_speaking_style_guide.md` in the repo root first and follow it.
